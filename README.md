## 1. 项目简介 (Introduction)

本项目是一款基于 **Java Swing** 开发的沉浸式 2D 角色扮演游戏（RPG）。玩家将扮演年轻的魔法师 `cyxxx`，肩负拯救世界的使命。游戏包含完整的剧情线，玩家需要在村庄、森林和洞穴等多个场景中自由探索，与 NPC 对话获取线索，在回合制战斗中击败怪物（如史莱姆、哥布林）以及最终 Boss 哥斯拉，最终让世界重见光明。

## 2. 核心功能 

- **多场景地图系统**：支持多个地图（村庄、森林、洞穴、雪山）的无缝切换。地图包含障碍物、传送点、NPC 交互区和 Boss 刷新点。每个地图都有独特的布局和挑战。
- **策略回合制战斗**：
  - 玩家拥有普通攻击、小技能（日之呼吸）、大招（爆裂魔法）等多种攻击手段，每种技能都有不同的MP消耗和伤害倍率。
  - 怪物拥有复杂的 AI，不同类型的敌人具备独特技能。Boss在低血量时会进入特殊状态（如哥斯拉的狂暴状态、暗影魔王的第二阶段）。
  - 战斗系统引入暴击机制，增加战斗的随机性和刺激感。
- **完善的 UI 交互**：
  - **主菜单**：支持新游戏、读取存档、音量调节和设置选项。
  - **战斗界面**：实时显示 HP/MP 进度条，支持战斗中打开背包使用道具，并提供详细的战斗日志系统。
  - **探索界面**：动态显示玩家位置、等级、经验值和当前地图信息。
- **双重持久化存储**：
  - **背包系统**：支持从文本文件读取初始配置，并能动态增删道具。战斗中可实时使用道具恢复HP/MP。
  - **全局存档**：利用序列化技术保存玩家当前坐标、属性、背包状态和所在地图，支持随时保存和加载游戏进度。
- **丰富的 NPC 与事件系统**：地图上分布着多种 NPC（村民、长老、商人、登山向导等），交互可触发随机对话；特定图块会触发随机遇敌或 Boss 战。
 - 使用WASD键控制角色移动
  - 查看地图信息：当前坐标、等级、生命值等
  - 与NPC交互（按N键位置）获取任务提示
  - 按Q键打开背包查看和使用道具
  - 按P键保存游戏进度

## 3. 系统架构与实现逻辑 

### 3.1 核心引擎与地图 (Game Engine & Map)

- **`GameMap.java`** (核心控制器):
  - **地图渲染**: 使用 `Map<String, String[][]>` 存储不同场景的二维数组地形。通过 `addGameMap()` 方法根据当前坐标动态重绘界面（玩家显示为 `^`，墙壁为 `#`）。
  - **移动逻辑**: 监听 `WASD` 键盘事件，处理碰撞检测（不能穿过 `#`）。
  - **场景切换**: 当玩家击败特定 Boss 或达到特定条件时，修改 `mapName` 并重置坐标以切换场景。
  - **事件触发**: 移动时检测地形符号，`N` 触发 NPC 对话，`.` 随机遇敌，`!` 触发 Boss 战。
  - **UI增强**: 引入进度条显示HP/MP状态，添加战斗日志文本区域，提供更直观的游戏信息反馈。

### 3.2 角色体系

- **`Character.java` (抽象基类)**: 定义了 HP, MP, Attack, Defense 等通用属性，作为所有游戏角色的基类。
- **`Player.java` & `Magician.java`**:
  - 实现了经验值系统 (`gainExp`) 和升级逻辑 (`levelUp`)，升级后自动提升属性上限。
  - 战斗技能通过委托模式调用 `BattleEngine` 进行计算，提高代码复用性和可维护性。
- **`Enemy.java` & 子类**:
  - **`Monster`**: 普通怪物，包含随机逃跑机制 (`flee`)。
  - **`Godzila` (Boss)**: 覆写 `actInBattle`，当 HP < 30% 时有概率触发双倍伤害的狂暴攻击。
  - **多样化敌人**: 包括史莱姆、哥布林、骷髅战士、兽人战士、火焰巨龙、森林守护者、洞穴巨兽、冰霜巨龙等，每种敌人都有独特的技能和行为模式。

### 3.3 数据持久化 

- **`GameSaver.java` & `SaveData.java`**:
  - 利用 `ObjectOutputStream` 将 `SaveData` 对象（包含 Player 对象、Inventory 对象、XY坐标、地图名）序列化到 `savegame.dat` 文件中，实现完整的游戏进度保存。
  - 支持游戏状态的完整保存和恢复，确保玩家可以随时中断并继续游戏。
- **`Inventory.java`**:
  - **混合存储**: 构造时通过 `Scanner` 读取 `Inventory.txt` 初始化道具；同时实现了 `Serializable` 接口，随全局存档一起被保存。
  - **回调机制**: 使用 `combatUIupdateCallback` 接口，确保在战斗中使用道具后，能即时刷新战斗窗口的 UI 数值。

### 3.4 交互界面 

- **`GameMenu.java`**:
  - 使用 `SwingUtilities.invokeLater` 确保线程安全，避免UI线程阻塞。
  - 实现了自定义图片按钮（`createImageButton`），并集成了背景音乐播放器 (`AudioPlayer`)。
  - 提供美观的主菜单界面，支持新游戏、读取存档、设置和退出等功能。
- **战斗界面**:
  - 采用现代化UI设计，使用进度条实时显示角色HP/MP状态。
  - 集成战斗日志系统，详细记录战斗过程中的所有动作和事件。
  - 提供直观的按钮操作界面，简化玩家操作流程。
